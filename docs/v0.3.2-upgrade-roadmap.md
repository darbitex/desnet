# DeSNet v0.3.2 — Upgrade Roadmap

**Compiled 2026-05-02 from R4 post-mainnet audit + live mainnet smoke findings.**

All items are **compat-safe** for an additive/body-only chunked upgrade on top of `v0.3.1-mainnet-deployed` (commit `09a2076`, on-chain upgrade_number 3, source_digest `89EBB7BAFE7AE180...`). Bundle for one chunked deploy via `multisig_publish_chunked_upgrade`.

## Mandatory fixes (LOW-bundle from R4 audit + live smoke)

### F1. factory: misleading `E_HANDLE_TAKEN` error code on view aborts
**R4 L1 — verified live on mainnet.** 5 view fns abort `E_HANDLE_TAKEN(0x3)` when semantic is "record not found":
- `factory::get_token_record(handle)`
- `factory::handle_of_token(token_metadata)`
- `factory::handle_of_owner(owner_addr)`  ← also has API design issue: keyed on pid_addr, not wallet_addr (see F1b)
- `factory::token_metadata_of_owner(owner_addr)`
- `factory::lp_staking_pool_of_owner(owner_addr)`

**Fix**: Replace `E_HANDLE_TAKEN` with existing `E_TOKEN_NOT_FOUND(=19)` in those 5 view fns. Body-only change, compat-safe.

**F1b (related):** `handle_of_owner(addr)` is misnamed — internally indexes by `owner_index` which is keyed on `pid_addr`, not wallet. So calling with wallet returns `E_HANDLE_TAKEN`; calling with pid_addr returns the handle. Either:
- Rename to `handle_of_pid(pid_addr)` — but breaks compat (rename = remove)
- OR keep name, add NEW companion `handle_of_wallet(wallet_addr)` that derives pid via `profile::derive_pid_address(wallet)` then forwards. Compat-safe.

### F2. governance: chunked-publish lacks empty-slot defense
**R4 L2.** `multisig_publish_chunked_upgrade` doesn't validate that all module slots in `code: vector<vector<u8>>` are non-empty before publishing. If chunk skipped (out-of-order user error), result has empty entries and `code::publish_package_txn` aborts with generic framework error instead of clear ours-error.

**Fix**:
```move
let i = 0; let n = vector::length(&code);
while (i < n) {
    assert!(!vector::is_empty(vector::borrow(&code, i)), E_INCOMPLETE_CHUNKS);
    i = i + 1;
};
```
Add right before `code::publish_package_txn(&pkg_signer, metadata, code)`. Add `E_INCOMPLETE_CHUNKS=23` const. Body-only.

### F3. governance: `cleanup_upgrade_staging` no event
**R4 L4.** State-reset is silent — observability gap.

**Fix**: Define + emit
```move
#[event]
struct UpgradeStagingCleanup has drop, store {
    multisig: address,
    timestamp_secs: u64,
}
```
At `cleanup_upgrade_staging` body. Compat-safe additive.

### F4. profile: `HandleRegistered` event — CLOSED
**R4 L5 DROPPED.** Re-checked source: `profile.move:142` defines `struct HandleRegistered has drop, store` and `profile.move:426` emits it. Live smoke also showed event fired in register_handle("desnet") tx. R4 finding was incorrect — no fix needed.

### F4b. amm: `compute_amount_out` missing `#[view]` annotation
**Live smoke #10.** `amm::compute_amount_out(reserve_in, reserve_out, amount_in)` is `public fun` but lacks `#[view]` → cannot be called via `/v1/view` REST endpoint (returns `function not marked as view function`). Frontend forced to make a tx OR call `quote_swap_exact_in` (which requires pool to exist on-chain — useless for ad-hoc CPMM quotes).

**Fix**: Add `#[view]` annotation. Pure fn, no `acquires`, deterministic — compat-safe annotation-only change.

```move
#[view]
public fun compute_amount_out(
    reserve_in: u64,
    reserve_out: u64,
    amount_in: u64,
): u64 { ... }
```

### F4c. amm: view-fn arg convention inconsistency (handle vs pool_addr)
**Live smoke gotcha.** Some views take `handle: vector<u8>`, others take `pool_addr: address`:
- handle-keyed: `reserves`, `lp_supply`, `quote_swap_exact_in`, `fee_buckets`
- pool_addr-keyed: `reserves_at`, `lp_supply_at`, `lp_fee_per_share`, `pool_locked`, `creator_pid`

Frontend integrators must memorize which-takes-which; misuse aborts `E_POOL_NOT_FOUND`. Recommend NEW companion fns providing the OTHER variant for each (additive, compat-safe). E.g., `reserves_by_handle` already exists as `reserves` — add `lp_fee_per_share_by_handle` etc.

### F5. amm: `Swapped` event `actor` field hardcoded `@0x0`
**Live smoke #14.** Swap events emit `actor: @0x0` instead of caller — breaks indexer attribution. Verified on tx `0xdc3a18761b635038210363f3c00d56af3e5113827629d201dc3759a7e168685f` (sender `0x47a3e1`, event `actor: 0x0`).

**Root cause**: `swap_exact_apt_in` / `swap_exact_token_in` are `public fun` (not entry) — no `&signer` parameter, can't recover actor. Generic `swap(_, _swapper: address, ...)` receives address but underscore-prefixed = unused.

**Fix options** (compat-safe, additive):
- **Option A (recommended)**: Add NEW pub fns `swap_exact_apt_in_actor(handle, actor, fa_in, min_out)` + `swap_exact_token_in_actor(...)`. Old fns still callable, deprecated path. Entry fns `swap_apt_for_token` / `swap_token_for_apt` route to new variant with `signer::address_of(caller)` as actor. Generic `swap(...)` rewires `_swapper` → `swapper` and propagates.
- **Option B (simpler)**: Move `event::emit(Swapped {...})` from inside `swap_exact_*` into the entry fns. Composable callers lose the event entirely (acceptable trade-off — they emit their own). Body-only inside entry fns; old pub fns lose their event emission. **Compat issue**: removing event emission from a public fn changes observable behavior — flag for review.

Recommend Option A.

## Deferred items from prior memory (still pending)

### F6. governance: auto-track `total_30d_emission` (Item 1)
Replace `update_total_30d_emission` setter with rolling-bucket counter hooked from `lp_staking::claim_internal`. Pool emission per claim accumulates into governance state. Eliminates manipulation surface (multisig setting arbitrary value). After F6 lands, F6b: neuter `update_total_30d_emission` (similar to F6 pattern of `update_desnet_fa_metadata` E_NEUTERED).

**Compat**: `update_total_30d_emission` body change to abort, add new fn `record_emission_for_window(amount)` friend-only called from lp_staking. Add new resource `Emission30dRollingBucket has key`. Both compat-safe additive.

### F7. governance: per-token rewards isolation (Item 2)
Separate `SmartTable<address, u64>` per FA addr in voter_history → voting_power = strictly DESNET-LP-earned, not mixed-token aggregate. Eliminates cross-token reward inflation of voting power.

**Compat**: Add new SmartTable field on VoterHistory (additive struct field on `key`-only struct = NOT compat-safe at struct level). Need parallel resource `VoterHistoryV2 has key` with per-token data, populated lazily. voting_power reads V2 if exists else falls back to V1.

### F8. governance: DAO chunked-upgrade variant
Sister fns to multisig variant: `dao_stage_upgrade_chunk(proposer)` + `dao_publish_chunked_upgrade(executor, proposal_id)` with hash check from approved+ratified proposal. Reuses `UpgradeStaging` resource (drop+key). Auth = anyone after proposal ratified + timelock passed.

**Compat**: Pure additive entries, compat-safe.

## Defensive cleanups (INFO from R4)

### F11. governance: `proposal_threshold/quorum_amount` overflow protection
`(state.total_30d_emission * BPS) / 10000` overflows u64 if `total_30d_emission > 2.6e15`. Bounded by economic reality but recommend hard-cap input in `update_total_30d_emission` (or after F6, in the auto-tracker fn).

```move
const MAX_30D_EMISSION: u64 = 1_000_000_000_000_000_000; // 1e18 raw
assert!(amount <= MAX_30D_EMISSION, E_AMOUNT_TOO_LARGE);
```

### F12. apt_vault, lp_emission, reaction_emission, factory: rename `_of_owner` views with mismatched semantic
Audit smoke revealed several views accept addresses that don't match their name's implication. Compile a full naming audit pass during F1b — affects ~8 view fns across modules. Each is body-only or NEW companion fn.

### F13. `D_SETTLE_THRESHOLD` consistency check across modules
Per memory, settle is permissionless explicit entry with dust protection. Verify same threshold used across `apt_vault::settle`, `giveaway::settle_giveaway`, future `handle_fee_vault::settle` (when re-introduced).

## Out-of-scope for v0.3.2

- **Sealing the package** (`upgrade_policy = immutable`): premature. Wait until at least F6 + F8 (auto-tracker + DAO chunked) so DAO can fully self-govern without multisig safety net.
- **`disable_multisig_upgrade()` on-chain call**: wait until DAO trusted in production (not before v0.3.4+). This is one-way.
- **Frontend wire-up**: separate workstream, not contract-side.
