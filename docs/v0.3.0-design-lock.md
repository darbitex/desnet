**Date:** 2026-05-01
**Status:** DESIGN LOCKED, implementation pending
**Supersedes:** v0.2.1 (lazy `seed_lp_staking_pool`)

## 1. Locked decisions

| # | Decision | Value |
|---|---|---|
| Pool gating | Open ONLY via `register_handle` (NOT permissionless) | hard |
| Composability | Shape-compatible with darbitex AMM (no Move dep) | external bots only |
| AMM fee | 10 bps, 100% to LP | no protocol cut |
| Locker behavior | Forever-lock (no withdraw fn exists) | irreversible |
| Lock + stake | Merged into single `desnet::lp_staking` module | custody pattern degenerate |
| Initial seed | Fixed 5 APT + 50M $TOKEN per pool | FDV ≈ 100 APT |
| Allocation | 50M pool / 50M reaction / 900M LP-emit (creator alloc dropped) | sums to 1B exactly |
| Hooks | Drop entirely | no V4-style callbacks |
| Pkg structure | Monolith `desnet` retained | friend-graph required |
| Pool seed source | User pays `handle_fee + 5 APT` flat (separate, not sliced) | UX clarity |
| Emission rate | 10 $TOKEN/sec = 1_000_000_000 raw/sec, FIXED at pool level | C-variant |
| C-variant | Per-staker share = `(their_shares / total_staked) × accumulated_per_share` | unchanged |
| Two stake types | Locked (atomic at register, creator's LP) + Free (open, withdrawable) | both feed voter_history + reference_gate |

## 3. New modules

### 3.1 `desnet::amm` (~800-1000 LoC)

**Purpose:** Purpose-built APT/$TOKEN pool. NOT a general-purpose AMM factory.

**Key types:**
```move
struct Pool has key {
    handle: vector<u8>,           // immutable, links pool ↔ handle
    apt_reserve: Object<FungibleStore>,       // APT held
    token_reserve: Object<FungibleStore>,     // $TOKEN held
    lp_supply: u128,              // total LP shares minted
    fee_bps: u16,                 // hardcoded 10, kept as field for view
    creator_pid: address,         // for downstream auth checks
}

struct LpShares has key, store {
    pool: Object<Pool>,
    shares: u128,
}
```

**Pool addr derivation:**
```move
public fun pool_address_of_handle(handle: vector<u8>): address {
    let seed = b"desnet::amm::pool::";
    vector::append(&mut seed, handle);
    object::create_object_address(&@desnet, seed)
}
```
Deterministic + handle-bound. Frontend can pre-compute. No registry lookup needed.

**Friend-only entry (called from factory at register_handle):**
```move
public(friend) fun create_pool_atomic(
    handle: vector<u8>,
    apt_in: FungibleAsset,         // exactly 5 APT
    token_in: FungibleAsset,       // exactly 50M $TOKEN
    creator_pid: address,
): LpShares acquires ...
```

**Public (permissionless) entries:**
```move
public entry fun swap_apt_for_token(caller, handle, amount_in, min_out)
public entry fun swap_token_for_apt(caller, handle, amount_in, min_out)
```

**Public views (shape-compatible with darbitex):**
```move
#[view] public fun quote_swap_exact_in(handle, amount_in, apt_to_token: bool): u64
#[view] public fun pool_address_of_handle(handle): address
#[view] public fun reserves(handle): (u64, u64)   // (apt, token)
#[view] public fun lp_supply(handle): u128
#[view] public fun fee_bps(handle): u16
```

**Math (constant product, no arb hook):**
- `amount_out = (reserve_out × amount_in_after_fee) / (reserve_in + amount_in_after_fee)`
- `amount_in_after_fee = amount_in × (10000 - 10) / 10000`
- LP fee accrues to reserve (no separate fee accumulator).

**Removed from darbitex parent:**
- arb module (`settle_profit`, arb_split, etc.)
- hooks (witness-gated callbacks)
- generic `Pool<A, B>` typed surface (we are APT/$TOKEN only — but views match shape so external aggregators see same field names)
- LP token-as-FA (we use `LpShares` resource, not standalone FA). External arb bots don't need to hold our LP — they just swap.

### 3.2 `desnet::lp_staking` (~300-400 LoC, merged lock+stake)

**Purpose:** Both forever-locked creator stakes AND open free-stakers earn from same emission reserve.

**Key types:**
```move
struct StakingPool has key {
    handle: vector<u8>,
    total_staked_shares: u128,
    rate_per_sec: u64,            // 1_000_000_000 raw = 10 $TOKEN/sec
    accumulated_per_share: u128,   // C-variant cumulative
    last_update_secs: u64,
    reserve_addr: address,         // pointer to LP_EMISSION_RESERVE in lp_emission
}

struct LockedPosition has key {
    pool: Object<StakingPool>,
    pid: address,                  // emission flows to object::owner(pid) at claim time
    shares: u128,
    last_acc_per_share: u128,      // C-variant snapshot
}

struct FreePosition has key {
    pool: Object<StakingPool>,
    staker: address,               // emission flows to this addr direct
    shares: u128,
    last_acc_per_share: u128,
}
```

**Friend-only entry (called from factory at register_handle):**
```move
public(friend) fun create_pool_and_lock(
    handle: vector<u8>,
    lp_shares: amm::LpShares,      // creator's LP shares (forever-locked)
    creator_pid: address,
): Object<LockedPosition>
```
**No `unlock` / `withdraw_locked` fn EXISTS.** Locked literally = no exit.

**Public entries (free-stake, anyone):**
```move
public entry fun stake_free(caller, handle, lp_shares: LpShares)
public entry fun unstake_free(caller, position_addr)
public entry fun claim_free(caller, position_addr)
public entry fun claim_locked(caller, position_addr)  // anyone can poke; emission goes to PID owner
```

**C-variant emission update:**
- Triggered on every stake / unstake / claim
- `now = timestamp::now_seconds()`
- `delta = now - pool.last_update_secs`
- `new_emission = delta × pool.rate_per_sec`
- `accumulated_per_share += (new_emission × 1e18) / pool.total_staked_shares`
- `last_update_secs = now`
- Per-position pending = `(pool.acc_per_share - position.last_acc_per_share) × position.shares / 1e18`

**Voter history wiring:**
- Both `claim_free` and `claim_locked` call `voter_history::record_reward_received(recipient_addr, amount)` after disburse
- This is the SOLE pathway for governance voting power generation

**ReferenceGate LP-stake check:**
- `reference_gate::check` queries `lp_staking::has_active_stake(addr, handle)` — returns true if either FreePosition or LockedPosition exists for that addr+pool
- Locked positions: addr = `object::owner(pid)` at check time

## 4. Atomic `register_handle` sequence

Single tx, friend-only, all-or-nothing:

```
1. profile::register_handle(caller, handle, ...)
   ├─ assert handle valid + not taken
   ├─ charge `handle_fee_apt(handle.length) + POOL_SEED_APT (5)` from caller
   │   - handle_fee → vault (vault::deposit_apt)
   │   - pool_seed (5 APT) → temp holding for step 5
   ├─ pid_addr = derive_pid_address(caller)
   ├─ mint PID NFT (Object NFT) at pid_addr, transfer to caller
   └─ call factory::spawn_token_atomic(handle, pid_addr, pool_seed_apt)

2. factory::spawn_token_atomic(handle, pid_addr, apt_for_pool):
   ├─ pkg_signer = governance::derive_pkg_signer()
   ├─ create $TOKEN FA via primary_fungible_store::create_fungible_asset
   │   - supply 1B × 10^8 = 1e17 raw
   │   - decimals 8
   │   - name/symbol from handle
   ├─ MINT FULL SUPPLY:
   │   - 50M (5%) → temp store for pool seed
   │   - 50M (5%) → reaction_emission::initialize_reserve
   │   - 900M (90%) → lp_emission::initialize_reserve
   ├─ create vault: apt_vault::initialize(handle, BurnRef<TOKEN>)
   ├─ call amm::create_pool_atomic(handle, apt_for_pool, 50M_token, pid_addr)
   │   → returns LpShares
   ├─ call lp_staking::create_pool_and_lock(handle, lp_shares, pid_addr)
   │   → returns Object<LockedPosition>
   ├─ initialize counters: history::init_counter, link::init_sync_set, etc.
   └─ emit RegisterHandleEvent
```

**Atomicity guarantee:** Move tx is all-or-nothing. Any abort in steps 1-2 reverts everything (no partial state).

## 9. Test plan (v0.3.0 ship gate)

**Unit (target ≥45):**
- `amm`: pool create, swap A→B and B→A, slippage rejection, zero-amount rejection, fee math, reserve invariants, pool addr determinism
- `lp_staking`: stake_free, unstake_free, claim_free, claim_locked (PID owner change between stake and claim), C-variant math under multi-staker concurrency, locked position immutability (no withdraw fn callable), emission depletion at reserve empty
- Pool addr collision: two handles → two pools, no overlap
- Allocation arithmetic: 50M + 50M + 900M = 1B exactly (no off-by-one)

**Integration (target ≥10):**
- Full register_handle atomic: all-or-nothing on each fail point (insufficient APT, $TOKEN spawn fail, pool create fail, lock fail)
- Vault buyback via in-house pool: deposit APT → swap to $TOKEN → burn
- Press royalty → vault → buyback chain
- PID transfer mid-stake: emission rights follow new owner
- Free stake + locked stake same pool: C-variant share math correct
- Reference gate LP-stake satisfaction via free or locked position

**Smoke (testnet):**
- 7 verbs end-to-end on freshly registered handle
- Vault settle with non-zero pool reserves
- Multi-handle parallel registration (no addr collision)

## 12. Open knobs (deferred to implementation)

- LpShares opaque struct vs explicit (u128, Object<Pool>) tuple
- Free-stake minimum (anti-dust DoS, e.g. 1 LP share min)
- Pool addr seed bytes exact form (`b"desnet::amm::pool::" + handle` proposed)
- LpShares::store ability — needed if we want positions to be transferable to other accounts
- C-variant denominator scale: 1e18 vs 1e12 (precision vs overflow headroom)
- Frontend disclosure UX for two position types (locked-yours-forever-but-emission-flows vs free-stake-withdrawable)
- voter_history weights: locked vs free position emission counted equally? (default: yes)
- Whether `claim_locked` is callable by anyone (poke) or only PID owner (saving gas vs forcing claims)
