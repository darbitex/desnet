## Dim 3 — Math

**u128 widening for multiplication (no u64 overflow):**
- `compute_amount_out`: u128 numerator, u128 denominator, cast back u64 only for result ✓
- `mint_lp_initial`: u128 product, u128 sqrt ✓
- `add_liquidity_internal`: u128 lp_from_X intermediates ✓
- `remove_liquidity_internal`: u128 X_out_u128 ✓
- `fee_per_lp_delta`: `(fee_amount as u128) × FEE_ACC_SCALE / lp_supply` — u128 × u128 = u256? Actually u128 × u128 may overflow. Let me verify:
  - max fee_amount = u64 max = ~1.8e19
  - FEE_ACC_SCALE = 1e18
  - 1.8e19 × 1e18 = 1.8e37 < u128 max (3.4e38). ✓ Fits.

**Division by zero:**
- `lp_supply > 0` guard before `fee_per_lp_delta` computation ✓
- `total_staked_shares > 0` (now `lp_supply > 0`) guard before `delta_per_share` in `update_pool` ✓
- `compute_amount_out`: `denominator = reserve_in × DENOM + amount_in_after_fee`. reserve_in checked > 0 in add_liquidity. amount_in checked > 0 in swap. denominator always > 0. ✓
- `lp_from_apt = apt_amount × lp_supply / apt_reserve_amt`: `apt_reserve_amt > 0` asserted ✓

**Truncation direction:**
- All divisions truncate (Move u64/u128 integer division). Direction always favors pool, never user. Standard. ✓
- Edge: pending fee = `(acc - last_acc) × shares / SCALE`. May truncate user share. Sum of all truncated pending ≤ accumulated bucket. No drain risk.

**Sqrt overflow:**
- `math128::sqrt(u128 product)` for initial LP. Max input = u64 × u64 = u128 max. Aptos stdlib handles. ✓

**Constants verified:**
- `FEE_BPS = 10` (10 bps) ✓
- `FLASH_FEE_BPS = 10` (10 bps, = LP swap fee per user spec) ✓
- `FEE_DENOM = 10000` ✓
- `MIN_INITIAL_LP = 1000` (anti-zero-share griefing) ✓
- `FEE_ACC_SCALE = 1e18` (precision/headroom balance) ✓
- `DEFAULT_RATE_PER_SEC = 1e9` (10 $TOKEN/sec at 8 dec) ✓
- `UNLOCK_FOREVER = u64::MAX` ✓

## Dim 4 — Reentrancy

**Flash loan lock (verified 6 sites):**
- `add_liquidity_internal`: `assert!(!pool.locked)` ✓
- `remove_liquidity_internal`: `assert!(!pool.locked)` ✓
- `swap_exact_apt_in`: `assert!(!pool.locked)` ✓
- `swap_exact_token_in`: `assert!(!pool.locked)` ✓
- `flash_borrow`: `assert!(!pool.locked)` then `pool.locked = true` ✓
- `flash_repay`: sets `pool.locked = false` at end ✓
- All swap/LP/flash entries gated. Round-trip lock invariant.

**Position object transferability (V3 NFT pattern):**
- Free / time-locked Position: created via `object::create_object(staker_addr)`, owner = staker, transferable.
- LockedPosition (creator): stored at pid_addr (move_to to existing PID Object via pid_signer). Not its own Object — moves with PID NFT. ✓ Forever-bound.

**Sealed LP for forever-lock (structural):**
- LockedPosition has `unlock_at_secs = u64::MAX`.
- `unstake` asserts `unlock_at != UNLOCK_FOREVER` BEFORE calling `amm::remove_liquidity_internal`. Aborts with `E_LOCKED_FOREVER`.
- amm pool reserves never returned for forever-locked shares. Locked shares contribute to lp_supply forever, accruing fees + emission to PID owner.
- LP shares are u128 counter (no FA store to seal physically). Counter immutability enforced by code path (no extract for forever case).

**Dispatchable FA reentrancy:**
- $TOKEN FA created via `primary_fungible_store::create_primary_store_enabled_fungible_asset` — standard FA, no dispatch refs generated. No on-deposit/on-withdraw hooks fire.
- APT FA at @0xa is standard framework FA — no dispatch hooks.
- LP FA: removed entirely (V3 NFT).
- No reentrancy via dispatchable FA possible in our flow.

**Cross-module reentrancy:**
- `lp_staking::claim_internal` reads amm accumulators, then calls amm::extract_fees_for_claim. Sequential within tx. No interleaved state change.
- `voter_history::record_reward_received` writes to centralized Registry. No callback to lp_staking.
- `lp_emission::pull_for_claim` withdraws from primary store. No callback.
- All cross-module flows linear, no reentry surface.

## Dim 5 — Edge Cases

**Zero amounts:** all entries assert > 0 ✓ (Dim 2)

**Empty reserves:**
- `add_liquidity_internal`: `assert!(apt_reserve_amt > 0 && token_reserve_amt > 0, E_INSUFFICIENT_LIQUIDITY)` ✓
- `swap`: amount_out > 0 guard ✓ (would compute 0 if reserves = 0; aborted)
- `remove_liquidity_internal`: apt_out > 0 && token_out > 0 ✓

**Pool not found:** all entries assert `exists<Pool>(pool_addr), E_POOL_NOT_FOUND` ✓

**Position not found:** all position entries assert `exists<Position>, E_POSITION_NOT_FOUND` ✓

**Single-sided liquidity:** NOT supported. add_liquidity requires both APT + TOKEN > 0. By design.

**Dust liquidity:**
- `MIN_INITIAL_LP = 1000` floor on first mint ✓
- Subsequent mints: `assert!(lp_minted > 0, E_INSUFFICIENT_LIQUIDITY)` ✓

**Empty fee bucket:**
- `extract_fees_for_claim`: `assert!(balance(apt_fees) >= apt_amount, E_INSUFFICIENT_FEE_BUCKET)` ✓
- In practice never aborts: per-position pending sum ≤ bucket (truncation favors pool).

**Forever-lock unstake attempt:** `unstake` aborts `E_LOCKED_FOREVER` ✓

**Time-lock unstake before unlock:** `unstake` aborts `E_LOCKED_NOT_YET_UNLOCKED` ✓

**Duplicate pool create:** `assert!(!exists<Pool>(pool_addr), E_POOL_ALREADY_EXISTS)` + factory registry uniqueness check + Aptos object framework deterministic addr collision ✓ (triple-protected)

**Duplicate handle:** factory registry `E_HANDLE_TAKEN` + profile `E_HANDLE_TAKEN` + profile `E_PID_ALREADY_EXISTS` ✓ (triple-protected)

**Flash loan dust amount (amount < 1000):**
- `fee = (amount × 10) / 10000 = 0` for amount < 1000
- Borrower can flash for free
- ⚠️ **L1 (LOW)** — Aave-acceptable (Aave 0% standard); document in WARNING

## Dim 6 — Cross-Module Interactions

**Atomic register_handle chain:**
```
profile::register_handle (entry, public)
├─ validate handle, charge handle_fee → fee_receiver, charge 5 APT → pool_seed_fa
├─ mint PID NFT (Profile resource at pid_addr via pkg_signer)
├─ register handle ↔ wallet in HandleRegistry
└─ factory::create_token_atomic(handle, pid_addr, &pid_signer, pool_seed_fa) [friend]
   ├─ validate handle (defense-in-depth), assert !exists in factory registry
   ├─ assert pool_seed_fa.amount == 5 APT exact
   ├─ pkg_signer = governance::derive_pkg_signer()
   ├─ create $TOKEN FA at deterministic addr, mint 1B supply
   ├─ split allocation: 50M pool / 50M reaction / 900M emission
   ├─ lp_emission::deploy(handle, token_meta, lp_emission_fa) [friend, returns reserve_addr]
   ├─ reaction_emission::deploy(handle, token_meta, reaction_fa) [friend]
   ├─ amm::create_pool_atomic(handle, pool_seed_apt, pool_seed_token, pid_addr) [friend, returns initial_shares u128]
   │   ├─ create Pool object at deterministic addr, sealed transfer
   │   ├─ create 4 child FungibleStores (apt_reserve, token_reserve, apt_fees, token_fees)
   │   └─ deposit reserves; set lp_supply = sqrt(apt × token)
   ├─ apt_vault::deploy(handle, token_meta, amm_pool_addr, pid_addr, burn_ref) [friend]
   ├─ lp_staking::create_pool_and_lock(handle, token_meta, reserve_addr, pid_addr, &pid_signer, initial_shares) [friend]
   │   ├─ assert !exists<StakingPool>, !exists<Position> at pid_addr
   │   ├─ assert pid_signer.addr == creator_pid
   │   ├─ create StakingPool object at deterministic addr (sealed transfer)
   │   └─ move_to(&pid_signer, Position { shares: initial_shares, unlock_at: u64::MAX, recipient_pid: creator_pid, ... })
   ├─ destroy mint_ref (fixed_supply forever)
   ├─ record TokenRecord in registry + 2 reverse indices (token_metadata, owner_addr)
   └─ emit TokenSpawned event (10 fields)
```

All steps atomic. Any abort reverts whole tx. PID NFT, $TOKEN, vault, pool, locked LP all created or none created. ✓

**Claim chain (PID owner triple-settle):**
```
lp_staking::claim(_caller, position_addr) [public, permissionless poke]
└─ claim_internal(position_addr)
   ├─ update_pool(pool_addr) — advance emission accumulator
   ├─ pending_emission = (acc - last_acc) × shares / SCALE
   ├─ amm::fee_per_lp(handle) — read fee accumulators (no mutation)
   ├─ pending_apt_fee = (fee_per_apt - last_apt) × shares / amm_scale
   ├─ pending_token_fee = (fee_per_token - last_token) × shares / amm_scale
   ├─ position.last_* updated to current
   ├─ recipient = if recipient_pid == @0x0 { object::owner(position) } else { object::owner(pid) }
   ├─ pending_emission > 0:
   │   ├─ lp_emission::pull_for_claim(reserve_addr, token_meta, pending_emission) [friend]
   │   ├─ primary_fungible_store::deposit(recipient, emission_fa)
   │   ├─ governance::derive_pkg_signer()
   │   └─ voter_history::record_reward_received(&pkg_signer, recipient, pending_emission)
   ├─ pending_apt_fee + pending_token_fee > 0:
   │   ├─ amm::extract_fees_for_claim(handle, pending_apt, pending_token) [friend, returns (apt_fa, token_fa)]
   │   ├─ primary_fungible_store::deposit(recipient, apt_fa)
   │   └─ primary_fungible_store::deposit(recipient, token_fa)
   └─ emit Claimed event
```

**Voting power source authentication preserved:**
- `voter_history::record_reward_received` is the SOLE pathway for voting power generation.
- Asserts `signer.addr == @desnet`.
- ONLY called from `lp_staking::claim_internal` (this is the SOLE call site post-V3 refactor — old `lp_emission::claim_with_governance` removed).
- `lp_staking` obtains pkg_signer via `governance::derive_pkg_signer()` (friend access).
- Cross-module auth chain: lp_staking (friend of governance) → governance (signer at @desnet) → voter_history (asserts @desnet). ✓ Chain integrity.

**Vault settle chain:**
```
apt_vault::settle(_caller, vault_addr) [public, permissionless]
├─ assert vault.apt_balance >= APT_SETTLE_THRESHOLD (0.1 APT)
├─ owner_addr = object::owner(pid_obj) — auto-resolves current PID owner
├─ split 50/50: buyback_amount + owner_amount
├─ apt_for_buyback = coin::extract → coin_to_fungible_asset
├─ amm::swap_exact_apt_in(handle, apt_fa, 0) → token_received
├─ fungible_asset::burn(&vault.burn_ref, token_received) — burn TOKEN
```

## Findings (Detail)

### M1 — `extract_fees_for_claim` not gated by flash lock (defense-in-depth)
**Severity:** MEDIUM
**Location:** `amm::extract_fees_for_claim` (line ~333)
**Issue:** During flash loan window, `lp_staking::claim` (permissionless poke) can trigger `extract_fees_for_claim` which pulls from fee buckets. Fee buckets are SEPARATE state from reserves, so no direct flash interaction. But the principle of "all state-mutating amm ops gated by lock" is partially broken.
**Risk:** Negligible in current V3 model (fee buckets isolated from reserve math). Theoretical concern: future upgrade that conflates fee bucket and reserve state could expose race.
**Fix:** Add `assert!(!pool.locked, E_LOCKED)` to `extract_fees_for_claim`.
**Status:** RECOMMENDED before mainnet (1-line addition).

### M2 — Position transferability (V3 NFT) not documented in user-facing docs
**Severity:** MEDIUM (UX clarity)
**Location:** `lp_staking::create_position` (line ~285)
**Issue:** Free/time-locked Position objects have ungated_transfer ENABLED by default (V3 NFT pattern). Not explicitly disabled. Lost-key/wrong-transfer = lost position.
**Risk:** User-facing UX confusion. Standard V3 NFT semantic but DeSNet docs should clarify.
**Fix:** Document in `WARNING` constant or frontend disclosure. NO code change.
**Status:** Documentation update only. Could be deferred to frontend.

### L1 — Flash loan dust amounts (< 1000 raw) charge 0 fee
**Severity:** LOW
**Location:** `amm::compute_flash_fee` (line ~657)
**Issue:** `fee = (amount × 10) / 10000`. For amount < 1000 raw, fee truncates to 0. Borrower can flash dust amounts for free.
**Risk:** Negligible — dust amounts have no economic value to flash-loan strategies. Aave-acceptable (Aave 0% standard).
**Fix:** Optional — `assert!(fee > 0 || amount == 0, E_ZERO_AMOUNT)` to enforce minimum economic value. OR document in WARNING.
**Status:** Document in WARNING (already noted in item 5 of disclosure).

### L2 — Fee bucket precision drift (V3 truncation)
**Severity:** LOW
**Location:** `amm` fee accumulator math
**Issue:** Each swap divides `(fee × SCALE) / lp_supply` (truncated). Each claim divides `((acc - last) × shares) / SCALE` (truncated). Cumulative truncation could leave dust in fee bucket forever.
**Risk:** Bounded dust accumulation — never exceeds 1 raw unit per claim per asset. Over 1M claims = max ~1M raw = 0.01 token at 8 dec. Negligible.
**Fix:** None needed. Standard V3 behavior.
**Status:** ACCEPTED.

### L3 — Doc-comment placement warnings (44 cosmetic)
**Severity:** LOW (cosmetic only)
**Location:** various — `///` placed before `#[view]`/`#[test_only]` pragmas
**Issue:** Move parser flags doc-comments not directly attached to language items.
**Risk:** None — compile + tests pass.
**Fix:** Convert `///` → `//` or move comment after pragma. Cosmetic only.
**Status:** DEFERRED.

### I1 — Voting power source pathway preserved post-V3 refactor
**Severity:** INFO
**Note:** `voter_history::record_reward_received` SOLE call site verified to be `lp_staking::claim_internal`. Old `lp_emission::claim_with_governance` removed. Voting power generation pathway = `lp_staking::claim` only. ✓

### I2 — Forever-lock structurally enforced
**Severity:** INFO
**Note:** For `unlock_at_secs == u64::MAX`, `unstake` aborts `E_LOCKED_FOREVER` BEFORE any extract path. amm::remove_liquidity_internal never callable for forever-locked shares. LP shares contribute to lp_supply forever, accruing yields to PID owner. ✓

### I3 — PID handle ↔ token name/symbol immutability
**Severity:** INFO
**Note:** factory creates $TOKEN FA with `name = symbol = handle` and NO `MutateMetadataRef` generated. Permanent inheritance. PID handle also immutable (no `update_handle` fn). Triple-protected uniqueness (factory registry + profile registry + FA deterministic addr). ✓

### I4 — Universal fee model (raw LP not possible)
**Severity:** INFO
**Note:** Post-V3 refactor, ALL LP exists in Position objects (no raw LP FA). Fee accumulator denominator = `lp_supply` (TOTAL). Every position earns fees uniformly. No "raw LP forfeits" semantic. Aligns with standard AMM economics (V2/V3 both have universal fee accrual). ✓
